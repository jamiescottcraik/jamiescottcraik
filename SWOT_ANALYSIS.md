### 2.3 Accessibility Paradox

| Issue | Evidence | Irony |
|-------|----------|-------|
| Heavy reliance on external images | Giphy, badge services, analytics GIFs | Screen readers get poor experience; violates accessibility ethos |
| No alt text documentation | Images lack `alt` attributes in source | Undeliverable on WCAG claims without manual audit |
| Animated/JS-heavy elements | Typing SVG, capsule render | May violate WCAG "no flashing >3x/sec" if not controlled |

**Action Required:**
1. **Audit all images** for alt text and decorative marking
2. Add **"Accessibility Statement"** section documenting actual compliance measures
3. Consider **reducing external dependency risk**: badge services go down, GIFs break
4. Add **Lighthouse accessibility scores** for README itself (via CI or manual audit)
